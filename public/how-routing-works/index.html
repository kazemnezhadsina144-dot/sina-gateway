How routing works

Routing is deterministic — rule order in code, not learned scoring. Each inquiry stores venture_route, route_rule_id, route_confidence, and route_reason.

Rule order (first match wins)

  1. campaign_founder_audit — UTM founder-audit → Founder Audit
  2. path_founder_audit — page path contains founder-audit → Founder Audit
  3. notes_founder_signal — notes mention founder audit / solo founder → Founder Audit
  4. identity_friend — Network identity → Personal
  5. identity_buildmatch — BuildMatch identity → BuildMatch
  6. identity_builder — Collaborator → Forge
  7. identity_investor — Investor → Noetfield
  8. intent_trust — Trust / risk goal → TrustField
  9. value_risk — Risk value → TrustField
  10. notes_trust_signal — trust / compliance notes → TrustField
  11. intent_invest — Invest goal → Noetfield
  12. partner_capital — Partner + capital → Noetfield
  13. partner_talent — Partner + talent → Forge
  14. intent_hire — Hire goal → SourceA
  15. execution_value — deal / project / lead value → SourceA
  16. default_strategy — fallback → Noetfield

Source of truth: ROUTING_RULES in src/gateway.js. Audit: npm run audit:routes.

What you see on intake
